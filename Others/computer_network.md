# 计算机网络
## 第1章 概述
- 互联网按工作方式可划分为边缘部分与核心部分。主机在网络的边缘部分，作用是进行信息处理。路由器在网络的核心部分，作用是按储存转发方式进行分组交换。
- 计算机通信是计算机中的进程之间的通信。计算机网络采用的通信方式是客户-服务器方式和对等连接(P2P)方式。
- 按作用范围，计算机网络可分为广域网(WAN)、城域网(MAN)、局域网(LAN)和个人区域网(PAN)。
- 计算机网络最常用的性能指标有：速率、带宽、吞吐量、时延、时延带宽积、往返时间、信道利用率。
- 网络协议是为进行网络中的数据交换而建立的规则。计算机网络的各层及其协议的集合称为网络的体系结构。
- 五层协议的体系结构由应用层、传输层、网络层、数据链路层和物理层组成。运输层最重要的协议是TCP和UDP协议，网络层最重要的协议是IP协议。
- 分组交换的要点：分组交换采用存储转发技术，每个分组都携带目的地址和源地址。分组交换不需要预先建立连接。
- 电路交换、报文交换和分组交换的优缺点。  
    电路交换：端对端通信质量因约定了通信资源获得可靠保障，对连续传送大量数据效率高。  
    报文交换：无需预约传输带宽，动态利用传输带宽，对突发式数据通信效率高，通信迅速。  
    分组交换：高效，灵活，迅速，可靠。
- 计算机网络各种类别的网络的特点。  
    广域网：远程、高速，是Internet的核心网。  
    城域网：城市范围，连接多个局域网。  
    局域网：校园、企业、机关、社区，范围较小。  
    个人区域网：个人电子设备，范围很小。
- 互联网的边缘部分和核心部分的特点。工作方式的特点。  
    边缘部分：由主机构成，用户直接使用；主机之间不是直接相连的，而是通过许多中间节点(路由器)来转接的。  
    核心部分：实现为多个网络互相连接而成的庞大网络；路由器之间采用分组交换技术。
- 网络体系结构采用分层次结构的原因。  
    1. 各层之间独立。
    2. 灵活性好。
    3. 结构上可分割。
    4. 易于实现和维护。
    5. 能促进标准化工作。
- 协议和服务的区别和关系。  
    协议：控制两个或多个对等实体进行通信的规则的集合。  
    在协议的控制下，两个对等实体间的通信使得本层能够向上一层提供服务。要实现本层协议，需要使用下面一层所提供的服务。下面的协议对上面的服务用户是透明的。  
    协议是“水平的”，即协议是控制两个对等实体进行通信的规则。服务是“垂直的”，即服务是由下层通过层间接口向上层提供的。上层使用所提供的服务必须与下层交换一些命令，这些命令在OSI中称为服务原语。
- 网络协议的三个要素及含义。
    1. 语法：即数据与控制信息的结构或形式。
    2. 语义：即需要发出何种控制信息，完成何种动作以及做出何种响应。
    3. 同步：即事件实现顺序的详细说明。

## 第2章 物理层
- 物理层的主要任务：确定与传输媒体接口有关的一些特性，如机械特性、电气特性、功能特性和过程特性。
- 数据通信系统的三大部分：源系统、传输系统、目的系统。源系统包括端点和发送器，目的系统包括接收器和终点。
- 传输媒体分为导引型传输媒体和非导引型传输媒体。
- 常见的信道复用技术有：频分复用、时分复用、统计时分复用、码分复用、波分复用。
- 宽带接入方法有非对称数字用户线(ADSL)、光纤同轴混合网(HFC)和FTTx。
- 物理层要解决的问题和主要特点。  
    主要问题：
    1. 物理层要尽可能屏蔽掉物理设备和传输媒体，通信手段的不同，使数据链路层感受不到差异，只考虑完成本层的协议和服务。
    2. 给其服务用户在一条物理的传输媒体上传送和接收比特流的能力，物理层应该解决物理连接的建立、维持和释放问题。
    3. 在两个相邻系统之间唯一地标识数据电路。

    主要特点：
    1. 涉及广泛，描述与传输媒体接口的机械，电气功能和规程特性。
    2. 物理连接很多，传输媒体种类很多，协议复杂。
- 规程与协议的区别。  
    在数据通信的早题，对通信所使用的规则都称为规程，后来具有体系结构的计算机网络开始使用协议这一名词，以前的规程其实就是协议，由于习惯，对以前制定好的规程有时仍常用旧的名词。
- 常用的传输媒体的特点。
    1. 双绞线
    2. 同轴线缆
    3. 光导纤维
    4. 无线电微波通信
- 比较ADSL,HFC和FTTx接入技术的优缺点。  
    ADSL用数字技术对现有的模拟电话用户线进行改造，使它能够承载宽带业务。成本低，易实现，但带宽和质量差异性大。  
    HFC具有很宽的频带，能够利用已经有相当大覆盖面的有线电视网。  
    FTTx可提供最好的带宽和质量，但线路和工程成本大。
- 在ADSL技术中，为什么不到1MHz的带宽中可以使得传送速率高达每秒几个兆比特？  
    靠先进的DMT编码，频分多载波并行传输，使得每秒传送一个码元相当于每秒传送多个比特。

## 第3章 数据链路层
- 链路是从一个节点到相邻节点的一段物理线路，数据链路是在链路的基础上增加必要的硬件和软件。
- 数据链路使用的信道主要有点对点信道和广播信道。
- 数据链路层传送的协议数据单元是帧。数据链路层的三个基本问题：封装成帧、透明传输和差错检测。
- 点对点协议PPP是数据链路层使用最多的一种协议，特点：简单；只检测差错，不纠正差错；不使用序号，也不进行流量控制；可同时支持多种网络层协议。
- PPPoE是为宽带上网等主机使用的链路层协议。
- 局域网的优点：具有广播功能，从一个站点可很方便地访问全网；便于系统的扩展和逐渐演变；提高了系统的可靠性、可用性和生存性。
- 共享通信媒体资源的方法：一是静态划分信道；二是媒体接入控制，又称为多点接入。
- 以太网采用的协议是具有冲突检测的载波监听多点接入CSMA/CD。协议的要点：发送前先监听，边发边监听，发现总线上出现碰撞就立刻停止发送。然后按照退避算法等待一段随机时间后再次发送。每一个站在自己发送数据之后的一小段时间内存在遭遇碰撞的可能性。以太网上的各站点都平等地争用以太网信道。
- 以太网的硬件地址(MAC地址)是适配器地址或适配器标识符，与主机所在的地点无关。源地址和目的地址都是48位。
- 以太网的适配器有过滤功能，只接受单播帧、广播帧或多播帧。
- 数据链路与链路的区别。  
    数据链路与链路的区别在于数据链路除链路外，还必须有一些必要的规程来控制数据的传输，数据链路比链路多了实现通信规程所需要的硬件和软件。
- 为什么要解决数据链路层三个基本问题？  
    基本问题：帧定界、透明传输和差错检测。  
    帧定界是分组交换的必然要求；透明传输避免消息符号与帧界定符号混淆；差错检测防止无效数据帧浪费后续路由上的传输和处理资源。
- 为什么PPP不使用帧的编号？PPP适用于什么情况？  
    只负责检错，不纠错；不会出现乱序；避免不必要的复杂性。  
    适用于线路质量不太差的情况。
- 局域网的主要特点。为什么局域网采用广播通信方式而广域网不采用？  
    特点：
    1. 共享传输信道。
    2. 地理范围有限，用户数量有限。
    3. 低层协议简单。
    4. 不单独设立网络层。
    5. 采用两种媒体访问控制技术。

    原因：局域网面对的问题是多源、多目的的连接管理。
- 什么是传统以太网？以太网的两个主要标准。  
    DIX Ethernet V2标准的局域网  
    DIX Ethernet V2和IEEE 802.3
- 以太网交换机特点。如何组成虚拟局域网？  
    以太网交换机为链路层设备，可实现透明交换。  
    虚拟局域网VLAN是由一些局域网网段构成的与物理位置无关的逻辑组。虚拟局域网协议允许在以太网的帧格式中插入一个4字节的标识符，称为VLAN标记，用来指明发送该帧的属于哪个虚拟局域网。

## 第4章 网络层
- 网络层向上只提供简单灵活的、无连接的、尽最大努力交付的数据报服务。网络层不提供服务质量的承诺，不保证分组交付的时限，所传送的分组可能出错、丢失、重复和失序。进程之间通信的可靠性由运输层负责。
- IP网是虚拟的，因为从网络层上看，IP网上一个统一的、抽象的网络。IP层抽象的互联网屏蔽了下层网络很复杂的细节，使我们能够使用统一的、抽象的IP地址处理主机之间的通信问题。
- 互联网上交付主机的两种方式：在本网络上的直接交付和到其他网络的间接交付。
- MAC地址是数据链路层wh物理层和使用的地址，而IP地址是网络层和以上各层使用的地址，是一种逻辑地址，从数据链路层看不见数据报的IP地址。
- 自治系统(AS)是在单一技术管理下的一组路由器。一个自治系统对其他自治系统表现出的是一个单一的和一致的路由选择策略。
- 路由选择协议：内部网关协议，如RIP和OSPF；和外部网关协议，如BGP-4。
- RIP是分布式的基于距离向量的路由选择协议，只适用于小型互联网。RIP按固定的时间间隔与相邻路由器交换信息。交换的信息是自己当前的路由表，即到达本自治系统中所有网络的最短距离，以及到每个网络应经过的下一跳路由器。
- OSPF是分布式的链路状态协议，适用于规模较大的互联网。OSPF只在链路状态发生变化时，才向本自治系统中的所有路由器泛洪发送与本路由器相邻的所有路由器的链路状态信息。所有路由器最终都能建立一个全网的拓扑结构图。
- IPv6变化：更大的地址空间；灵活的首部格式；改进的选项；支持即插即用；支持资源的预分配；IPv6首部改为8字节对齐。
- 虚拟专用网VPN是利用公用的互联网作为本机构各专用网之间的通信载体。VPN内部使用互联网的专用地址。所有通过互联网传送的数据必须加密。
- 使用网络地址转换NAT技术可以在专用网络内部使用专用IP地址，仅在连接到互联网的路由器使用全球IP地址。
- 软件定义网络SDN不是要改变网络的功能，而是一种新的体系结构，要点：基于流的转发；数据层面与控制层面分离；控制层面用软件实现，集中式控制；可编程的网络。
- 网络层向上提供的两种服务及优缺点。  
    面向连接的虚电路服务和无连接的数据报服务。  
    优点：提供服务质量的承诺。  
    缺点：路由器复杂，网络成本高。
- IP地址的特点。
    1. 是一种分等级的地址结构。
    2. 是标志一个主机或路由器和一条链路的接口。
    3. 用转发器或网桥连接起来的若干个局域网仍为一个网络，这些局域网具有同样的网络号。
    4. 所有分配到网络号的网络、范围很小的局域网和广域网都是平等的。
- 最大传送单元MTU？和IP数据报首部的字段关系？  
    IP层下数据链路层所限定的帧格式中数据字段的最大长度，与IP数据报首部的总长度字段有关系。
- IGP和EGP的区别。  
    IGP：在AS内使用的协议，力求最佳路由。  
    EGP：在不同AS间使用的协议，力求较好路由。
- RIP, OSPF和BGP的特点。  
    |协议|RIP|OSPF|BGP|
    |--|--|--|--|
    |网关协议|内部|内部|外部|
    |算法|距离矢量|链路状态|距离矢量|
    |传送方式|UDP|IP|TCP|
- VPN及其特点和优缺点和类别。  
    VPN：虚拟专用网络  
    优点：减少运营成本，降低连接成本。
    缺点：可靠性和性能不直接控制。  
    类别：PPTP VPN, L2TP VPN, IPSec VPN, SSL VPN, MPLS VPN
- NAT的优缺点？NAPT的特点？  
    优点：节省公有IP地址；地址重叠时，提供解决方法；网络发生变化时，避免重新编址。  
    缺点：无法进行端到端的IP跟踪；很多应用层协议无法识别。

## 第5章 运输层
- 运输层提供应用进程间的逻辑通信，并不是真正在两个运输层之间直接传送数据。使应用进程看见的是两个运输层之间端到端的逻辑通信信道。
- 网络层为主机之间提供逻辑通信，而运输层为应用进程之间提供逻辑通信。
- 运输层主要协议：TCP和UDP。都有复用和分用以及检错的功能。TCP协议的逻辑信道相当于全双工的可靠信道。UDP协议的仍是不可靠信道。
- 进程之间通信要知道对方的IP地址和端口号。
- UDP的主要特点：无连接；尽最大努力交付；面向报文无拥塞控制；支持一对一、一对多、多对一和多对多的通信；首部开销小。
- TCP的主要特点：面向连接；每一条TCP连接只能是点对点；提供可靠交付的服务；提供全双工通信；面向字节流。
- TCP用主机的IP地址加端口作为TCP连接的端点。这样的端点叫套接字。
- 流量控制就是让发送方的发送速率不要太快，让接收方来得及接收。
- 在某段时间内，若对网络中某一资源的需求超过了该资源所能提供的可用部分，网络性能变坏。这叫拥塞。拥塞控制就是防止过多的数据注入到网络中，这样可以使网络中的路由器或链路不至于过载。
- 流量控制是一个端到端的问题，是接收端抑制发送端发送数据的速率，以便使接收端来得及接收。拥塞控制是一个全局性的过程，涉及所有的主机、所有的路由器，以及与降低网络传输性能有关的所有因素。
- TCP拥塞控制算法：慢开始、拥塞避免、快重传和快恢复。在网络层，也可以使路由器采用适当的分组丢弃策略减少拥塞的发生。
- 运输连接三个阶段：连接建立、数据传送和连接释放。
- 运输层在协议栈中的地位和作用。运输层和网络层通信区别。  
    运输层处于面向通信的最高层，用户功能的最底层，向上提供服务。  
    运输层为应用进程提供端到端的逻辑通信，网络层为主机之间提供逻辑通信。  
    各种应用进程之间通信需要可靠或尽力而为两类服务质量，必须由运输层已复用和分用的形式加载到网络层。
- 为什么有些程序采用不可靠的UDP？  
    VOIP：由于语音信息具有一定冗余度，但对传播时延的变化较敏感。有差错的UDP数据报在接收端被之间抛弃，TCP数据报出错则会引起重传，可能带来较大的时延扰动。
- 为什么说UDP是面向报文，TCP是面向字节流的？  
    UDP对应用程序的报文在添加首部后就向下交付IP层。  
    TCP对报文数据块则视为无结构的字节流，可拆分，维持各字节。
- UDP和IP的不可靠程度是否相同？  
    不相同。UDP用户数据报的校验和既检验首部又检验载荷部分，而IP数据报的校验和仅检验IP数据报的首部。UDP用户数据报的校验和还增加伪首部。
- 流量控制和拥塞控制的最主要区别？发送窗口的大小取决于流量控制还是拥塞控制？  
    流量控制是在一条TCP连接中的接收端采用的措施，用来限制发送端发送报文的速率。  
    拥塞控制是控制发送端发送速率，防止互联网某处产生过载。

## 第6章 应用层
- 应用层的许多协议是基于客户服务器方式的。客户是服务请求方，服务器是服务提供方。
- 域名服务器分为根域名服务器、顶级域名服务器、权限域名服务器和本地域名服务器。
- 万维网WWW是一个大规模的、联机式的信息储藏所，可以非常方便地从互联网上的一个站点链接到另一个站点。
- 万维网的客户程序向互联网中的服务器程序发出请求，服务器程序向客户程序送回客户所要的万维网文档。在客户程序主窗口上显示出的万维网文档称为页面。
- 在万维网中用来搜索的工具叫做搜索引擎。搜索引擎大体上可以划分为全文检索搜索引擎和分类目录搜索引擎。
- 套接字是应用进程和运输层协议之间的接口，是应用进程为了获得网络通信服务而与操作系统进行交互时使用的机制。

## 第7章 网络安全
- 计算机网络上的通信面临的威胁有被动攻击和主动攻击。主动攻击的类型有更改报文流、拒绝服务、伪造初始化、恶意程序等。
- 数字签名功能：报文鉴别，即接收者能够核实发送者对报文的签名；报文的完整性，即接收者确信所收到的数据和发送者发送的完全一样而没有被篡改过；不可否认，即发送者事后不能抵赖对报文的签名。
- 防火墙是一种特殊编程的路由器，安装在一个网点和网络的其余部分之间，目的是实施访问控制策略。防火墙里面的网络是可信网络，外面的网络是不可信网络。防火墙的功能：阻止；允许。
- 防火墙技术分为：网络级防火墙，用来防止整个网络出现外来非法侵入；应用级防火墙，用来进行访问控制。

## 第8章 互联网上的音频/视频服务
- 多媒体信息的重要特点：多媒体信息的信息量大；对时延和时延抖动均有较高的要求。含有边传输、边播放的特点。
- 互联网提供的音频/视频服务类型：流式存储音频/视频，用户通过互联网边下载、边播放。流式实况音频/视频，其特点是在发送方边录制、边发送，在接受时也是要求能连续播放。交互式音频/视频，如互联网电话或互联网电视会议。
- 流媒体就是流式音频/视频，其特点是边下载、边播放。若使用专门的软件，可以将其储存在硬盘上。
- 为什么RTP协议同时具有运输层和应用层的特点？  
    实时运输协议RTP为实时应用提供端到端的运输，不提供任何服务质量保证。  
    从应用角度看，RTP将代码分组交给UDP，是一个应用层协议。  
    向多媒体应用程序提供服务，可以看成一个UDP之上的一个运输层子层协议。
